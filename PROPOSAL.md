# Bangla Hope Children’s Sponsorship Service
## Sponsorship Program Management System Proposal

**Organization:** Bangla Hope  
**Purpose:** Digital system to manage sponsorship programs for orphans, destitute children, and vulnerable families.

---

# 1. Project Overview

The **Bangla Hope Sponsorship Program Management System** is a centralized desktop-based system designed to manage student records, sponsorship data, school programs, financial assistance, and reporting.

The system will replace or improve the current **MS Access (2019)** workflow and provide a more structured, scalable, and maintainable solution.

The system will support:

- Student data management
- Sponsorship tracking
- Financial assistance tracking
- Reporting and document generation
- Program administration

All modules will operate **within one integrated system** used in **a single office environment**.

---

# 2. Current Situation

### Existing Infrastructure
- Good computer infrastructure
- Current system: **Microsoft Access 2019**
- IT Support: **2 IT staff located in Hili town**

### Limitations of Current System
- Limited scalability
- Difficult reporting and data management
- Manual data handling
- Limited user role control
- Data duplication across programs

### Goal of New System
To build a **structured, centralized desktop system** that:

- Improves data organization
- Simplifies sponsorship management
- Provides reliable reporting
- Enables multi-user role management
- Maintains historical records

---

# 3. System Architecture

The system will be a **single integrated desktop application** with modular program sections.

Modules represent different **Bangla Hope programs**, including:

1. Orphanage Residential Program
2. Boarding School Sponsorship Program
3. Village Schools Program
4. Higher Study Loan Program
5. Employee Children Support Program
6. Archive / Drop List System

# 6. Student Identification System

A student may participate in **multiple programs during their lifetime**.

Rules:

- Each program assigns a **new ID number**
- The **old ID is preserved for historical reference**
- Historical data remains searchable

This ensures **accurate record tracking across programs**.
